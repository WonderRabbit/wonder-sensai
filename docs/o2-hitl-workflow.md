# O2 — HITL(Human-in-the-Loop) 워크플로우

> 사람 개입 지점·메커니즘. sLLM 한계를 사람이 보완(언제·어떻게·무엇을).

## 개입 지점 (언제)

| 지점 | 이유 | 사람 역할 |
|---|---|---|
| decomposition | 고차원 추론 → sLLM 품질 상한 | task 분해 주도/승인(템플릿 보조) |
| 비즈니스 규칙·불변조건 승격 | 코드 추론 환각 위험 | 추출된 규칙 교차검증 후 제약 승격 |
| 일관성 게이트 위반(D4) | 컨벤션/비즈니스 위반 설계 | 수용/수정/재설계 결정 |
| 게이트 에스컬레이션 | steps 초과·할루시네이션 루프 | 개입/조정/중단 |
| AS-IS 범위 결정 | 전 시스템 비용 | 변경 영역 중심 범위 결정 |
| 입학 게이트 승인 | 신규 스킬/codegraph 편입 | 증거 검토 후 승인/거절 |
| 테스트 트리거 | per-run 옵션 | "이번에 테스트 실행?" 결정 |

## 메커니즘 (어떻게)

- **opencode TUI**: lead가 작업 중단 + 상태 보고(블록 사유·옵션) → 사용자 입력 대기 → 응답 시 재개.
- **에스컬레이션 보고서**: 구조화(무엇이 막혔는지·시도·옵션·권장) — R6 체크리스트 양식.
- **기록**: 요청-응답을 `.omo/evidence/` 또는 trace(출처=`elicited`)에 로그.

## 컨펌 게이트 (run.md F0-F5 연계)

- **soft**: F1→F2, F2→F3, F4→F5 (같은 축 내) — 확인 후 진행, 생략 가능(자동 진행 옵션).
- **hard**: F3(AS-IS accept), F5(미션 accept) — **사람 명시적 컨펌 필수**. verdict·시각·사유를 `progress.json`에 기록.

## 원칙

- 사람은 **판단/승인/에스컬레이션**만; 실행(수집/투영/검증)은 하네스.
- 자동화 가능 결정(근거 기반)은 사람 개입 없이 → 근거 부족/고판단만 사람.
- HITL 빈도 최소화(sLLM 자율 영역 확대는 입학 게이트 후).
