# 목표

후보(도구가 수집한 패턴)와 확인된 사실(도구로 재확인)을 분리한다. 모든 주장은 `path:line` 근거를 가져야 하고, 근거가 없으면 `UNKNOWN`으로 보존한다. 근거 없는 연결·발명·축약을 금지한다.

## 원칙 (불변)

1. **후보 ≠ 사실** — `ast-grep`/`rg` 후보 수집 결과는 사실이 아니다. 확인 전에는 후보로만.
2. **모든 사실은 ≥1 `path:line` 근거** — 정규 trace의 `evidence`에 역대조.
3. **근거 없으면 `UNKNOWN`** — 삭제·발명 금지. 후속 확인이 필요한 정상 결과.
4. **모순·모순성·모호성 보존** — 하나로 축약하지 않는다. 충돌은 `conflict`, 복수 후보는 `ambiguous`/`many_to_many`, 동적만 결정되면 `unresolved`.
5. **이름 유사도·가장 가까운 후보로 빈 연결을 채우지 않는다.**
6. **주석·문자열·예제는 근거가 아니다** — 코드 구조와 값 전달로 확인된 것만.

## 절차

1. 도구(`fd`/`rg`/`ast-grep`/`jq`/`yq`)로 후보를 수집한다. 후보에는 `path:line`을 붙인다.
2. 각 후보를 직접 도구로 재확인한다 — 변수 정의·리터럴·호출 관계·값 전달.
3. 확인된 것만 사실로 확정(`status: exact`). 미확인은 `UNKNOWN`/`unresolved`/`ambiguous`/`conflict`로 보존.
4. 모순·모호성은 각각 그대로 기록(축약 금지).
5. peer가 반환한 경로·줄은 lead가 직접 재확인 후 사용.

## 실패 폐쇄

- 근거 없는 주장을 하지 않는다. 한 사례를 관찰했다고 전체 규칙으로 일반화하지 않는다.
- 비언어 문자열(주석·문서) 일치는 근거가 아니다.
- 검증 실패 시 기존 산출을 덮어쓰지 않고 오류·다음 확인만 보고한다.
